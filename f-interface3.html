This project was created by using processing 4 (4.3) combined with the Arduino. This is a interactive two player game that helps to spread knowledge about waste management while playing a game.

 

Using Processing where the one of the players press keys ('R', 'G', or 'B') to match the letter with circle color. It utilizes serial communication to receive signals from an Arduino to change the circle's color. The game tracks points and missed, if you miss three in a raw you will loose the game. In the end you will receive a short message to influence the player to manage waste properly.

Wiring Instructions:

1. Connect Ultrasonic Sensor to Arduino:

VCC -> 5V on Arduino

Trig -> Digital Pin 2 on Arduino

Echo -> Digital Pin 3 on Arduino

GND -> GND on Arduino

2. Connect Regular LEDs to Arduino:

Connect the anode (+) of each LED to a digital pin on the Arduino through a resistor (330-500 ohms).

Connect the cathode (-) of each LED to GND on the Arduino.

Ensure to use appropriate resistors to limit current flow and protect the LEDs.

Use breadboard as the base to connect different components and jump wires to connect them.

Developing the Arduino Code

Variable Initialization:

  • unsigned long previousMillis = 0;

This variable is used to store the previously recorded time in milliseconds. It's initialized to 0 at the start of the program.

  • unsigned long currentMillis = 0;

Similar to previousMillis, this variable of type unsigned long is used to store the current time in milliseconds. It's also initialized to 0.

  • unsigned long elapsedTime = 0;

This variable is utilized to calculate the elapsed time between measurements. It's initialized to 0 at the beginning of the program.

  • long duration, distance;

These variables are used to hold the duration (the time it takes for the ultrasonic signal to travel and return) and the calculated distance in centimeters, respectively. They are declared but not initialized explicitly at the beginning because their values will be assigned later in the code based on the sensor readings.

Setup Function:

Initializes serial communication.

Sets the pin modes for the sensor trigger and echo pins as well as for the LED pins.

Loop Function:

Measures the distance using the ultrasonic sensor.

Calculates the elapsed time since the previous measurement.

Based on the elapsed time, it adjusts delays to change the frequency of sensor readings.

Prints the waste type detected on the Serial Monitor for debugging purposes.

Distance Measurement and LED Activation:

Determines the distance measured by the ultrasonic sensor and activates the appropriate LED based on the distance thresholds set (activategreen(), activatered(), activateblue()).

Developing the Processing Code

Serial Communication Setup and Variables

 

  • Serial Communication Setup :

import processing.serial.* - Imports the serial library to facilitate communication with external devices.

 

  • Variable Initialization :

myPort : Declares a variable for serial communication.

circleFillColor : Sets the default color for the circle.

colorKeys : Defines an array containing characters ('R', 'G', 'B') as keys for colors.

points : Tracks the player's points.

missedColors : Counts the number of missed colors.

gameStarted : Boolean flag to indicate if the game has begun.

circleSize : Initial size for the displayed circle.

Setup Function

setup() - Function

size(800, 600) - Sets the dimensions of the window to 800x600 pixels.

myPort = new Serial(this, "COM3", 9600) - Initializes serial communication with the specified port ("COM3") and baud rate (9600).

Draw Function

 

draw() - Function

background(220) - Clears the window and sets the background color to light gray.

if (gameStarted) { ... } - Displays the circle if the game has started.

textAlign(CENTER) - Sets the text alignment to center for score, missed count, and instructions.

displayInfo1();, displayInfo2();, displayInstructions(); - Calls functions to display score, missed count, and game instructions.

if (missedColors >= 3) { gameOver(); } - Checks if the player has missed three colors and triggers the game over function.

while (myPort.available() > 0) { ... } - Reads data from the serial port.

wasteType = myPort.readStringUntil('\n'); - Reads a string from the serial port until a newline character is encountered.

Processes the received data to change the circle's color based on the signals ("GREEN" "RED" or "BLUE") received from an external device (Arduino).

Key Press Handling

 

keyPressed() Function:

Handles keyboard inputs from the player.

Checks if the game has started and compares the pressed key to the expected color keys.

Updates points, missed colors, and triggers functions to change color and size or end the game based on player input.

Display Functions

Display Functions (displayInfo1(), displayInfo2(), displayInstructions()):

displayInfo1(): Displays the player's points.

displayInfo2(): Displays the count of missed colors.

displayInstructions(): Displays game instructions including how to play and the consequences of missing colors.

Game Logic Functions

 

pickRandomColor():

Randomly selects a color ('R', 'G', or 'B') and changes the displayed circle's color accordingly.

getColorChar():

Determines the character corresponding to the current circle's color.

randomizeSize():

Randomly changes the size of the displayed circle within a defined range to add variation to the game.

gameOver():

Displays a "Game Over!" message and stops the game loop when the game ends due to missing colors.
