This interactive project harnesses the power of Processing 4 (version 4.3) to create a dynamic graphical user interface (GUI). Unlike conventional games, this project revolves around crafting a visual interface using mouse and keyboard inputs. The primary aim is to construct a user-friendly environment for manipulating shapes and colors, fostering an engaging experience.

The fundamental objective of this GUI project is to design an interface that allows users to modify graphical elements and their attributes. It empowers users to control shapes, sizes, and colors while visualizing changes in real-time. The ultimate goal is to facilitate an intuitive interface that provides an enjoyable, interactive experience.

  • Understanding the Code Structure

ControlP5 Library Integration:

  • ControlP5 is an essential library in Processing used for creating graphical user interfaces. It offers a suite of tools to easily implement and manage various UI elements such as checkboxes, sliders, buttons, text fields, and more. This library simplifies the process of designing interactive elements within a Processing sketch.

 

Initialization in Setup():

  • The setup() function is where the Processing sketch initializes necessary components. This includes defining the canvas size using size(), initializing the ControlP5 library with new ControlP5(this), and setting up various interface elements like checkboxes, sliders, and buttons using ControlP5's methods.

 

Draw Loop in Draw():

  • The draw() function continuously redraws the canvas, updating the interface elements and shapes. It includes essential commands like background() to refresh the canvas with a specified color (backgroundColor), stroke() to set the outline color of shapes, and conditional statements (if-else) to draw either rectangles or ellipses based on the value of the drawRectangle boolean variable.

 

Shape Drawing Functions:

  • The drawRectangle() and drawEllipse() functions define how shapes (rectangles and ellipses) are drawn. These functions utilize Processing's shape-drawing commands like rect() and ellipse(), positioning them at the center of the canvas (width / 2, height / 2) and using the shapeSize variable for dimensions.

 

Event Handling Functions:

  • controlEvent(ControlEvent theEvent) handles interactions with ControlP5 elements. It specifically listens for events from the "toggleShape" checkbox, updating the drawRectangle boolean variable based on the checkbox state (checked or unchecked).

 

Slider and Button Functions:

  • adjustSize(float newSize) adjusts the shapeSize variable based on the slider's value. The changeColor() function randomly generates a new background color upon clicking the "Change Background Color" button using random().

 

Variable Declarations:

  • Declarations like boolean drawRectangle, int shapeSize, and int backgroundColor are positioned outside the functions to maintain their state across the sketch. These variables control the drawing behavior, shape size, and background color, respectively.

Connecting to IOT

Requirements

  • Adafruit IO Account: Sign up at Adafruit IO .
  • Username and Key: You'll need your Adafruit IO username and an active AIO Key.
  • Feed: Create a feed on Adafruit IO to send data and receive.
  • Dashboard: Using a toggle button you can send data to your feed.

Connect processing Code to Adafruit IO (IOT).

Initialization:

  • ioUsername, ioKey, and feed variables hold your Adafruit IO username, AIO Key, and feed name, respectively.
  • adafruitIOUrl constructs the URL to fetch data from the specified Adafruit IO feed using your username and feed name.
  • setup() Function:
  • setup() is a function in Processing that runs once at the beginning of the program.
  • It sets the canvas size to 200x200 pixels and sets the background color to dark gray (background(50)).
  • fill(200) sets the fill color to a lighter gray for text display.
  • Data Retrieval from Adafruit IO:
  • loadStrings() is a Processing function used to load data from a URL as an array of strings.
  • It fetches data from the constructed adafruitIOUrl by appending the AIO Key as a query parameter x-aio-key.
  • The data fetched from Adafruit IO is stored in the lines array.
  • Processing the Retrieved Data:
  • The code then checks if the lines array is not null.
  • If data is successfully fetched (lines != null), it enters a loop (for (String line : lines)) to iterate through each line in the retrieved data.
  • Inside the loop, it prints each line of the data to the console using println().
  • Handling Failure:
  • If the data retrieval fails (lines == null), it prints a message to the console indicating the failure to retrieve data from Adafruit IO.
