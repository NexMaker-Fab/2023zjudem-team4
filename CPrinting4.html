In this blog, We will share with you the results of an experiment we conducted to compare the infill pattern of the Flash Forge Guider IIs 3D printer.
Infill patterns are the internal structures that fill the empty space inside a 3D printed model. They affect the strength, weight, time, and cost of the print, as well as the aesthetics.
Choosing the right infill pattern for a specific project can be tricky, as there are many factors to consider and trade-offs to make.
So we decided to conduct a comparison experiment to test and evaluate five different infill patterns on the same model using the same settings and material.

 

 

Modeling
The first step is to make the model. Here we created the model by ourself to keep things interesting.
This is the model that we used in this experiment. We will use one filling pattern to fill one part and so on.

 

The model is then converted into a format that the 3D printer can understand, such as STL or OBJ. In our case we used the .stl file.

 

 

Slicing
The second step is to divide the digital model into thin horizontal layers, or slices, using a slicing software. The slicing software also generates instructions for the 3D printer, such as the speed, temperature, and direction of the print head. The instructions are saved as a file, such as G-code or X3G, that can be sent to the 3D printer.
We used FlashPrint software to slice and prepare the models for printing. FlashPrint is a powerful and user-friendly slicing software that offers a wide range of features and settings to optimize the printing process and ensure the best results.

Printing
The third step is to load the material, or filament, into the 3D printer and start the printing process. The 3D printer follows the instructions from the slicing software and deposits the material layer by layer on a build platform, creating the object. Depending on the size and complexity of the object, the printing process can take from minutes to hours or even days.

G Code Reading
G code is a programming language that tells a 3D printer how to move and what to do. It consists of a series of commands that specify the coordinates, speeds, temperatures, and other parameters for the printing process.
Each command starts with a letter, such as G or M, followed by some arguments, such as numbers or words. For example, G1 X10 Y10 F3000 means move to the point (10, 10) at a speed of 3000 mm/min.

Here we tried to explain some common g code syntax we have simplified for you.

G00: Rapid positioning. Moves the machine at maximum speed to a specified point.

G01: Linear interpolation. Moves the machine in a straight line at a specified feed rate.

G02: Circular interpolation clockwise. Moves the machine along a circular arc in a clockwise direction.

G03: Circular interpolation counterclockwise. Moves the machine along a circular arc in a counterclockwise direction.

G04: Dwell. Pauses the machine for a specified time.

G20: Inch units. Sets the units to inches.

G21: Millimeter units. Sets the units to millimeters.

G28: Return to home. Moves the machine to the home position.

G90: Absolute positioning. Sets the coordinates to be relative to the origin.

G91: Incremental positioning. Sets the coordinates to be relative to the current position.

M00: Program stop. Pauses the program execution and allows the operator to jog or alter the system.

M01: Optional stop. Checks the condition of a checkbox and pauses the program execution if it is checked. Allows the operator to jog or alter the system.

M02: Program end. Ends the program execution and returns to the beginning of the program.

M03: Spindle on clockwise. Starts the spindle rotation in a clockwise direction.

M04: Spindle on counterclockwise. Starts the spindle rotation in a counterclockwise direction.

M05: Spindle stop. Stops the spindle rotation.

M06: Tool change. Changes the tool with the one in the currently called up position.

M07: Coolant on (mist). Turns on the mist coolant.

M08: Coolant on (flood). Turns on the flood coolant.

M09: Coolant off. Turns off the coolant.

M30: Program end and reset. Ends the program execution, returns to the beginning of the program, and resets all modal states.
